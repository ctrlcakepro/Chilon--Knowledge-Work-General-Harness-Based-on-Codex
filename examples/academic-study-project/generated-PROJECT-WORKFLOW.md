# PROJECT-WORKFLOW Example: Academic Study Project

## 项目目标

把这个项目作为高校学习、心理学阅读和课程写作的协作底座，用稳定流程产出摘要、提纲、复习笔记、比较表、汇报结构和修改稿。

## 主要工作流

### 1. 教材与课程材料整理

适用场景：教材章节、课堂讲义、课程 PDF、阅读材料。

默认动作：

1. 识别材料主题和章节结构。
2. 提炼主线和核心概念。
3. 区分定义、理论、例子和研究证据。
4. 输出复习版笔记。
5. 补充易混点和可能考点。

### 2. 心理学理论比较

适用场景：比较两个理论、概念或研究取向。

默认动作：

1. 先列比较维度。
2. 用表格呈现差异。
3. 总结各自适用场景。
4. 给出一句判断。

### 3. 论文与作业写作

适用场景：课程论文、短评、作业题、读书报告。

默认动作：

1. 先解释题目要求。
2. 给出可行中心论点。
3. 生成提纲。
4. 标注每部分要解决的问题。
5. 用户要求时再扩写正文。

### 4. 汇报与展示

适用场景：课堂汇报、组会展示、slides 结构。

默认动作：

1. 判断听众和时长。
2. 确定主信息。
3. 设计讲述顺序。
4. 给逐页结构。
5. 需要时补讲稿。

### 5. 文本修改与检查

适用场景：作业修改、段落润色、逻辑检查、语言压缩。

默认动作：

1. 先给总体判断。
2. 指出主要问题。
3. 提供修改版。
4. 必要时解释改动。

## 默认收尾

每次任务完成后，给出一个自然下一步，例如：扩写、压缩成复习版、做表格、生成汇报结构或检查是否满足作业要求。
